# Human Writing

寫文章時避免 AI 的罐頭味，檢測文章時揪出 AI 生成的痕跡。

## 觸發時機

- 用戶要求寫文章、部落格、專欄
- 用戶要求檢測/分析一段文字是否像 AI 寫的
- 用戶說「幫我寫得更有人味」

## 模式一：寫作輔助

寫作時，主動避開這些 AI 味句型：

### 禁用清單

| 類型     | 禁用句型                                                         |
| -------- | ---------------------------------------------------------------- |
| 假轉折   | 不是…而是…、不僅…更…                                             |
| 風險緩衝 | 在某種程度上、可以說是、某種意義上                               |
| 假深度   | 然而我們也必須意識到、關鍵在於、這意味著                         |
| 罐頭結尾 | 總的來說、綜合以上、讓我們共同期待、需要我們每個人共同努力       |
| 空洞修飾 | 多維度、深遠影響、賦能、範式轉換、不可忽視的重要性               |
| 中立廢話 | 這是一個值得深思的問題、每個人可能有不同看法、這取決於多方面因素 |

### 替代策略

1. **用具體取代抽象**
   - 壞：「失敗是成功的母親」
   - 好：「創業失敗那天，我躲在便利商店吃冷掉的御飯糰，眼淚掉在包裝袋上」

2. **敢站邊、有立場**
   - 壞：「這件事各有利弊，我們應保持理性」
   - 好：「我認為這根本是在搞破壞，沒有第二種解讀」

3. **用生活譬喻**
   - 壞：「這段程式碼邏輯混亂」
   - 好：「這段程式碼亂得像我家那堆永遠理不開的耳機線」

4. **允許不確定**
   - 壞：「這個現象值得我們深思」
   - 好：「老實說，這件事我到現在還沒完全想清楚」

5. **允許矛盾和轉變**
   - 好：「我以前很討厭這種說法，但後來發現我錯了」

6. **結尾可以懸著**
   - 不需要總結，寫到沒話就停
   - 好的結尾：「到現在，我還不確定這是不是好事」

### 寫作前必問

向用戶索取：

- 你的個人經驗或故事？
- 你對這件事的真實立場？（不是「客觀看法」）
- 有沒有生活中的比喻可以用？

## 模式二：AI 味檢測

分析文章時，檢查三個核心問題：

### 檢測清單

| 問題   | AI 味指標                  | 人味指標                           |
| ------ | -------------------------- | ---------------------------------- |
| 具體性 | 沒有時間、地點、人物、數字 | 有具體細節、場景描寫               |
| 情緒感 | 句句通順、完美平衡         | 有不太通順但真實的句子、有明顯情緒 |
| 立場性 | 兩邊討好、不敢站邊         | 明確立場、獨到觀點、敢得罪人       |

### 檢測輸出格式

```
## AI 味檢測報告

**AI 味指數**: X/10（10 = 純 AI 味）

### 發現的 AI 句型
- [引用原文] → 問題：[說明]

### 缺少的人味元素
- [ ] 具體時間/地點/人物/數字
- [ ] 個人經驗或故事
- [ ] 明確立場
- [ ] 生活化譬喻
- [ ] 情緒表達

### 改寫建議
[針對最嚴重的 2-3 處給具體改法]
```

## 注意

- 這些句型「出現」不代表一定是 AI 寫的，人也會用
- 關鍵是「密度」：AI 文章這些句型會密集出現
- 人類文章有「凸點、凹洞、偏執與破綻」，這是特徵不是缺點
