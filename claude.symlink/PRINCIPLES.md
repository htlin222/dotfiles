# Principles (Compressed)

## Directive

Evidence > assumptions | Code > documentation | Efficiency > verbosity

## SOLID

- **S**: Single responsibility
- **O**: Open/closed
- **L**: Liskov substitution
- **I**: Interface segregation
- **D**: Dependency inversion

## Design

DRY | KISS | YAGNI | Composition>Inheritance | SoC | Loose coupling | High cohesion

## Senior Mindset

- **Decisions**: Systems thinking, long-term, stakeholder balance, risk calibration
- **Errors**: Fail fast+explicit, never suppress, preserve context, graceful degradation
- **Testing**: TDD, pyramid (unit>integration>E2E), tests=docs, comprehensive coverage
- **Deps**: Minimize, security monitor, justify, semver
- **Perf**: Measure first, perf=feature, monitor, resource aware
- **Logs**: Purposeful, structured, context rich, no secrets

## Quality

- **Functional**: Correctness, reliability, completeness
- **Structural**: Organization, maintainability, debt
- **Performance**: Speed, scalability, efficiency
- **Security**: Vulnerabilities, access, data protection

## AI-Dev

- **Generation**: Context-aware, incremental, pattern recognition, framework aligned
- **Tools**: Capability map, parallel ops, fallbacks, evidence-based selection
- **Errors**: Proactive detect, graceful degrade, context preserve, auto recover
- **Testing**: Comprehensive, risk-based, automated, user-centric
